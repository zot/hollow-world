## Implementation Notes

**Button State Logic:**
```typescript
// Nope button only enabled when hasUnsavedChanges === true
updateButtonStates() {
    const yepBtn = container.querySelector('#yep-character-btn');
    const nopeBtn = container.querySelector('#nope-character-btn');

    if (this.hasUnsavedChanges) {
        yepBtn.disabled = false;
        nopeBtn.disabled = false;
    } else {
        yepBtn.disabled = true;
        nopeBtn.disabled = true;
    }
}
```

**Hash Recalculation:**
```typescript
// After loading original, recalculate hash to reset change tracking
const original = await CharacterStorageService.getCharacter(id);
this.originalCharacterHash = await calculateCharacterHash(original);
this.character = original;
this.hasUnsavedChanges = false;
```

**Complete UI Refresh:**

- Skills, benefits, equipment all refreshed

- Hash reset means subsequent edits will be detected properly
- Browser back button NOT affected (history unchanged)
- Change detection continues running (250ms polling)
